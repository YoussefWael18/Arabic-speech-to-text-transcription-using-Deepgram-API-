# Arabic-speech-to-text-transcription-using-Deepgram-API-# Arabic Speech-to-Text Pipeline (Deepgram Nova-3)

A complete framework for evaluating contemporary cloud-based Speech-to-Text (STT) inference on the Arabic Speech Corpus using Deepgram's Nova-3 AI model, featuring custom tooling for large-scale dataset batching and a graphical user interface for results evaluation.

---

## 1. Source Code

All source code for the inference automation and the dataset visualization interface is housed in this repository.

### Key Components:
- `API/speech_to_text_streaming.py`: The robust batch inference script that systematically matches localized `.wav` files with their transcripts and executes the Deepgram API hooks.
- `API/results_gui.py`: The graphical Streamlit web application that lets researchers playback audio samples while reviewing predicted strings vs. base truth strings side-by-side.

*Please review the codebase in this GitHub repository for full implementation details.*

## 2. Dataset Description

**The Arabic Speech Corpus**
The dataset utilized in this project is an open-source Arabic Speech Corpus specifically designed for speech recognition and synthesis research. 

- **Language:** Modern Standard Arabic (MSA).
- **Format:** High-quality Studio recording `.wav` files.
- **Transcriptions:** Ground truths are provided deterministically in an `orthographic-transcript.txt` file utilizing **Buckwalter Transliteration** (a strict ASCII-based character map used to reliably represent Arabic without unicode formatting issues). 
- **Structure:** Features a rigorously separated "main" structure as well as a distinctly isolated "test set" for unbiased validation.

## 4. Experiments

Our experiment centered on feeding the unstructured `.wav` components of the Arabic Speech dataset directly into Deepgram's state-of-the-art **Nova-3** endpoint. 

- **Procedure:** The `speech_to_text_streaming.py` engine iterates over the dataset indices, generates a POST buffer of the WAV payload synchronously, and waits for Deepgram to calculate the Arabic phonetic matches. 
- **Scale:** The experiments were run using batched `--limit` constraints (e.g. 5 or 20 samples per block) during initial development to ensure mapping alignment before evaluating the broader dataset.
- **Encoding Map:** As part of the experiment validation, we factored the discrepancy between the dataset's Buckwalter Transliteration ground-truths (e.g., `waraj~aHa`) versus the Nova-3's natively exported Unicode Arabic script.

## 5. Evaluation Results

The outcomes of the STT inferencing are dumped organically into `API/results.csv`.

**Qualitative Observations:**
- **Accuracy Overview:** Deepgram Nova-3 displayed exceptional intelligence in transcribing spoken Arabic seamlessly into formal standard contextual representations.
- **Encoding Fidelity:** The STT outputs mapped identically with the linguistic intents encoded by the Buckwalter ground truths.
- **Failures & Anomalies:** A few edge cases returned localized API HTTP `400 Bad Request` or localized SSL handshake timeout warnings, demonstrating the necessity of robust error catching during large scale batch HTTP processing.

- **Ground Truth (Buckwalter):** `mim~aA qado yu&ad~iy <ilaY taraAjuEi masaAHaAti Alo>anohaAri`

## 6. Demo Interface

To simplify the evaluation of our experimental results, we developed an asynchronous Streamlit browser GUI (`results_gui.py`) which acts as an active dashboard. 

### GUI Features:
- Dropdown selection of localized evaluation audio inputs.
- Playable native `<audio>` widgets for listening to test files immediately.
- Real-time side-by-side rendering comparing the deterministic Buckwalter Text alongside Deepgram's Deep Learning Arabic generation.

### 📝 Local Setup Guide

If you wish to clone this repo and run it locally:
1. `cd API`
2. `.\venv\Scripts\activate` (or setup your own python virtual environment)
3. Ensure an `.env` exists with `DEEPGRAM_API_KEY="your_api_key_here"`
4. Run `python speech_to_text_streaming.py` to test inference.
5. Run `streamlit run results_gui.py` to boot up the User Interface.
